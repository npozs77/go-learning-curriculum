# Chapter 01: Modules, Build & Project Structure

**Learning Goal:** After this chapter, you will be able to create a Go module, manage dependencies, build and cross-compile binaries, run tests, and write proper documentation comments.

**Prerequisites:** None — this is the first chapter.

## Lessons

1. [Lesson 1: go.mod and Module Declaration](lesson1-go-mod/) — Understand how go.mod declares your project as a module
2. [Lesson 2: go.sum and Dependency Verification](lesson2-go-sum/) — Learn how cryptographic hashes protect your dependencies
3. [Lesson 3: The internal/ Directory Pattern](lesson3-internal-pattern/) — Use compiler-enforced privacy for your packages
4. [Lesson 4: Building Go Binaries and Cross-Compilation](lesson4-cross-compilation/) — Build self-contained binaries for any platform
5. [Lesson 5: Running Tests with go test](lesson5-go-test/) — Run tests with Go's built-in test toolchain
6. [Lesson 6: Makefile Usage for Common Project Tasks](lesson6-makefile/) — Create one-word shortcuts for common Go commands
7. [Lesson 7: Godoc Comment Conventions](lesson7-godoc/) — Write documentation that Go tools understand

## Exercises

1. [Exercise 1: Initialize a Go Module](exercises/exercise1-init-module/) — Read and parse your own go.mod file
2. [Exercise 2: Add a Dependency](exercises/exercise2-add-dependency/) — Simulate dependency management with deduplication
3. [Exercise 3: Cross-Compile a Go Binary](exercises/exercise3-cross-compile/) — Generate cross-compilation commands for multiple platforms
